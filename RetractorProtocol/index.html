💡 Why Two Specs for One Feature?
A single mechanical feature — locking force — controls two independent failure modes: too weak (safety floor) and too stiff (release ceiling). Both require separate acceptance criteria. Both will be tested. This is the foundation for the TOST procedure in Station 7B.

📐 Delta Must Be Clinical, Not Statistical
The non-inferiority margin is not the predicate mean. It is the minimum clinically meaningful threshold — derived from clinical evidence, not from what's convenient to measure. FDA will ask for the source document.

⚖️ TOST: Two One-Sided Tests
Equivalence ≠ two-tailed testing. TOST = two separate one-sided tests, each with its own H₀. Equivalence is declared only when both null hypotheses are independently rejected. FDA expects this structure for a bounded performance claim.

Footnote: With multiple simultaneous tests, a Bonferroni correction would split α — not required for this single bounded claim, but relevant for multi-endpoint submissions.

You completed a full V&V hypothesis testing workflow for a surgical retractor locking mechanism — the same structure used in actual 510(k) submissions.

It started with the FMEA, where you identified that one physical feature — locking force — controls two independent failure modes. That's why you needed two acceptance criteria, not one.

You set those criteria using clinical data: a surgical literature threshold for the lower bound and a human factors study for the upper bound. Delta must be clinically justified, not statistically convenient.

You built directional hypotheses where H₀ always represents the failure scenario — the thing you're trying to disprove. You chose α and power based on FMEA severity ratings, and calculated N using the standard formula with documented inputs.

The test results showed p = 0.003 (lower bound) and p < 0.001 (upper bound). Both null hypotheses were rejected. This is TOST — the correct structure when making a bounded equivalence claim.

In a real submission, every step appears in the Design History File: the FMEA, acceptance criteria with clinical rationale, statistical analysis plan, and test report with the TOST conclusion.
