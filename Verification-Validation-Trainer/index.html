Verification vs Validation Trainer
The back half of the design-controls waterfall — where Product Mapping Mastery leaves off at design inputs, this trainer picks up.
Verification
“Did we build it right?”
Design verification confirms that design outputs meet design input requirements. Methods include tests, inspections, and analyses against documented acceptance criteria.
21 CFR 820.30(f) · ISO 13485:2016 §7.3.6
Validation
“Did we build the right thing?”
Design validation ensures the device conforms to defined user needs and intended uses — testing production units (or equivalents) under actual or simulated use conditions, including software validation and risk analysis where appropriate.
21 CFR 820.30(g) · ISO 13485:2016 §7.3.7
The Design-Controls Waterfall
Verification closes the loop from outputs back to inputs; validation closes the loop from the finished device back to user needs.

Grounding: 21 CFR 820.30(f) design verification and 820.30(g) design validation; ISO 13485:2016 §7.3.6 and §7.3.7. Note: FDA's Quality Management System Regulation (QMSR, effective Feb 2026) incorporates ISO 13485:2016 by reference — the 820.30(f)/(g) concepts taught here map directly to ISO 13485 §7.3.6–7.3.7. This is a framework trainer: classifications reflect the design-controls logic of those clauses, and real programs document their own V&V rationale in the design plan.
